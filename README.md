# Fitbit Personal Health Analytics & Dashboard

A comprehensive local Data Engineering and Data Science pipeline to analyze, visualize, and benchmark personal health data exported from Fitbit.

This project uses **Python** for the ETL process (cleaning, merging, statistics) and **Node.js/Chart.js** for an interactive local dashboard.

## ðŸš€ Features

- **Multi-Format Ingestion:** Handles mixed granulaties (intra-day JSONs vs daily CSVs) and timezones automatically.
- **Data Cleaning:** Auto-deduplication of records, timezone normalization, and outlier handling.
- **Custom Metrics:**
  - **Readiness Score (Z-Score):** Combines Sleep Quality and Resting Heart Rate (RHR) deviation to recommend training intensity.
  - **Health Benchmarks:** Statistical analysis to determine your personal "Normal", "Optimal", and "Warning" zones.
- **Interactive Dashboard:** A modern, dark-mode web UI (Vite + Chart.js) featuring:
  - Dual-axis trends (Weight vs Calories).
  - Interactive drill-down on specific dates.
  - Dynamic KPI cards with RHR, Sleep, and Readiness status.
- **Daily Briefing:** A CLI script for a quick text-based check-up.

## ðŸ“Š Logic Behind the "Readiness Score"

The Readiness Score is calculated using **Z-Scores** (Standard Deviations from your personal mean):

- `Readiness = (Sleep Z-Score) - (RHR Z-Score)`
- **Logic:** Better sleep (+ score) and lower resting heart rate (- score inverted) result in a higher readiness.
- **Interpretation:**
  - `> 1.0`: **Peak Condition** (Push hard)
  - `-1.0 to 1.0`: **Normal** (Train as usual)
  - `< -1.0`: **Recovery Needed** (Focus on rest)
